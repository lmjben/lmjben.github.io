# Linux sed 命令在 mac 上使用的坑

在 mac 上使用 sed -i 参数时会报错，解决方法如下：

sed -i 's/xxx/yyy/g' xxx.txt
1

修改为：

sed -i '' -e 's/xxx/yyy/g' xxx.txt
1

# Git 默认对文件名大小写不敏感

如果只修改一个文件名称的大小写，执行 git status 时是检查不到的，如果要配置大小写敏感，需要执行git config core.ignorecase false
